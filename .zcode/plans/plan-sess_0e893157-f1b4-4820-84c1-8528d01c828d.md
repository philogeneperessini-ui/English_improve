# 对话记录持久化 + 输入框优化

## 需求
1. 对话切换/离开后回来，消息清空了 → 希望能恢复上次对话，并可选"新对话"，还能翻看历史对话记录
2. 输入框太窄，语音识别出较多文字时不便查看 → 输入框变高（多行）

## 数据层改动

### `src/lib/types.ts` — 新增对话类型
```ts
export type ConversationRecord = {
  id: string;
  scenario: ScenarioId;        // daily/travel/work/ideas
  createdAt: string;
  updatedAt: string;           // 每次发消息更新，用于排序
  messages: ChatMessage[];     // 完整消息（含 correction/source）
};
```
`ScenarioId`、`ChatMessage` 目前定义在 conversation-screen.tsx 里，把这两个类型提到 types.ts 共享。

### `src/lib/storage.ts` — 升级到 DB version 2，加 conversation store
- `openDB(DB_NAME, 2, upgrade)`：version 2 时新建 `conversation-records` store（keyPath: id）
- 新增 `saveConversation`（put，更新或插入）、`listConversations`（按 updatedAt 降序）、`deleteConversation`、`deleteAllConversations`
- 现有 practice store 不动（upgrade 里判断 contains 避免重建）

> 注意：idb 的 upgrade 回调里能用 `database.oldVersion` 判断，只在新版本时建新 store，不影响旧数据。

## 组件改动 `src/components/conversation-screen.tsx`

### 1. 持久化逻辑
- `ConversationScreen` 接收一个 `conversationId?` prop（从父组件传入，恢复指定对话）；不传时表示新对话
- 挂载时：
  - 有 `conversationId` → 从 IndexedDB 读出该对话，恢复 `messages`/`scenario`
  - 无 `conversationId` → 新建（现有的 firstMessage 逻辑）
- 每次 `messages` 变化后（debounce 或发送后），调 `saveConversation` 落库
- `changeScenario` 改为：切换场景时**不清空**，而是弹"开始新对话/继续"选择（见下）
- 组件卸载前做最后一次保存

### 2. 历史/新对话交互
- 顶部场景选择栏下方加一个紧凑的**历史对话入口**（如一个可点击的横条/按钮"历史对话(N)"）
- 点击展开一个**历史对话列表抽屉/面板**：
  - 每条显示：场景标签 + 首条消息预览 + 更新时间 + 消息数
  - 点击某条 → 加载该对话（调父组件切换 conversationId）
  - 每条带删除按钮
- 右上角现有的"重新开始"按钮（RefreshCw）改为弹出确认：**继续当前对话 / 开始新对话**
  - 当前对话会被保存，开始新对话时清空消息并落库一条新记录

### 3. 输入框变高
- textarea 的 `rows` 从 1 改为 3，`min-h-11` 改为 `min-h-24`，保留 `max-h-28` 限制 + 自动滚动
- 这样识别出多行文字时默认就能看到更多，不必手动展开

## 父组件改动 `src/components/speaking-app.tsx`
- 维护一个 `activeConversationId` 状态（默认 undefined = 新对话）
- `ConversationScreen` 接收 `conversationId` + 一个 `onConversationChange(id)` 回调
- 进入对话 tab 时，默认加载最近一条对话（若有），否则新建

## 不改动
- API 路由、ASR、题库、评价、练习记录全不动
- IndexedDB practice store 结构不变

## 验证
- `npm run lint` + `npm run build` 通过
- 浏览器实测：对话几轮 → 切到其他 tab → 切回来 → 消息还在；能从历史列表选不同对话；输入框变高能看更多字
